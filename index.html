Accreted Intelligence
AI that remembers what worked.

A model that scores 90% on a benchmark today scores 90% tomorrow. It doesn't learn from deployment, doesn't track which outputs led to good outcomes, doesn't remember last week's mistake. It generates intelligence and throws it away. Accreted intelligence is the bet that this is temporary: move learning out of model weights and into scored external state — where judgment compounds from contact with reality, and the model is a replaceable processor.

The problem

Recall is not judgment.

The central unsolved problem is not generating outputs — it is accumulating reliable judgment across time. The field treats it as a memory problem and answers with RAG, vector stores, and fine-tuning. But recall returns previously seen information; judgment knows what worked — that an approach succeeded here, failed there, holds in this context but not that one. In reinforcement learning this is the credit assignment problem, and deployed systems don't solve it: they emit outputs, observe nothing about consequences, and reset.

Recall is the easy part. Scoring is harder. Binding scoring to action is the part everyone skips — and it is the part that decides whether intelligence compounds.
The thesis

The intelligence is in the genome, not the organism.

DNA does not think or plan. It is a scored record of what survived contact with reality. Organisms are temporary processors: they express the genome, meet the environment, and either propagate the successful patterns or are eliminated. Most AI inverts this — the model is treated as the intelligence, state as supporting material. Accreted intelligence flips it back.

1State comes first

The scored state determines what the system knows, avoids, and tries next — not the model's momentary disposition.

2Reality does the scoring

Memory rises or falls on observed outcomes — not designer guesses, not the system's own confidence.

3The processor is replaceable

Intelligence lives in the substrate, so the reasoning model can be swapped without losing what was learned.

4Failure is scored knowledge

Bad outcomes debit the units that aligned and stay retrievable as warnings. A system that forgets its mistakes becomes overconfident fast.

The system — acc4

Two verbs. One memory. Four links.

acc4 is a working kernel for the thesis: a Recursive Language Model over a late-interaction scored-token memory. The entire reasoner interface is two verbs, and every non-trivial call closes four links — owner intent, the cited memory that shaped the work, the act, and the outcome that closes it. Citation is the credit edge, not decoration: the memory that gets cited is the memory that earns or loses score when the outcome lands.

acc4_retrieve(query | image)

The only read. Peeks the scored memory by MaxSim — natively multimodal: text and images encode to the same late-interaction tokens, so a text memo can answer an image query (ColPali-style).

acc4_act(runtime, input)

Does anything. solve recurses on a sub-goal — the tree of recursive solves is the decomposition; no planner, no task graph. exec runs sandboxed code that can itself recurse over the memory. register stores reusable code as scored tokens — no privileged lane.

The memory

The token is the atom.

Every entity — a knowledge memo, a runtime, an owner fact, an image — is stored as an ordered set of per-token vectors. No lossy pooled embedding; retrieval ranks by MaxSim (ColBERT), and the aligned token pairs are themselves the citation. That much is published commodity. What acc4 adds is a learned-evidence reweighting: each document token carries a Bayesian Beta posterior summarizing how its past alignments fared, and tokens reality has repeatedly confirmed pull harder than fresh ones.

score(q, d) = Σi maxj [ sim(qi, dj) · g(πj) ]

πj — token j's outcome posterior · the exact form of g and its calibration are proprietary

When a commitment closes, credit does not smear across a whole document — it flows to the specific tokens that aligned during retrieval. The update is surprise-gated (confirming outcomes move a token little, surprising ones move it more) and reality-gated: an outcome validated by reality — a real reply, a passing test, a world result — carries full weight, while a self-graded outcome credits only at a deliberately weak prior. The system structurally refuses to compound from its own belief.

Honest status

What's proven, what's open.

acc4 is a working single-host research kernel — a running system with live, scored mechanisms. It trains nothing; it is a memory-and-loop kernel around replaceable models. Two different model families read and extend the same substrate through one interface today, and neither owns the judgment. The claims are deliberately bounded, and the largest ones remain open:

  • Substrate-on vs. off lift — the single most important missing experiment: same tasks, scored substrate enabled vs. disabled, measure the delta. Falsifiable; not yet measured.
  • Scale — exact MaxSim over per-token multivectors grows with the substrate; the right pruning regime without losing unit-level credit is open.
  • Credit under concurrency — attributing outcomes to the right memory when many workers act in parallel is harder than in a serial loop.
  • Dependable replay — turning a successful trace into a runtime that replays reliably, without re-reasoning each time.
  • The authority boundary — the line between "may draft" and "may send" must stay hard as the system touches live accounts.
A system that compounds judgment must invest as heavily in governance, credit honesty, and boundary enforcement as in scoring and accumulation.
